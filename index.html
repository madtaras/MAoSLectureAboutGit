Бекапи

Git

Ініціалізація Git в папці проекту

$ git init

Для того, щоб почати використовувати Git треба ініціалізувати

Всі повторюють на своїх пк ті операції, які я показую

Налаштуємо Git клієнт

$ git config --global user.name "Billy Everyteen" 
$ git config --global user.email "your_email@example.com"
Тепер налаштуємо Git клієнт

Перевірка статусу

$ git status

Океей. Репозиторій ми створили імя налаштували, а шо далі?

створимо файл перший файл проекту index.html.

Збереження змін

$ git add [шляхи до файлів, які ми хочемо зберегти] 
$ git commit -m "[назва коміта]"
Тепер треба зберегти зміни.

Для цього потрібно виконати дві команди.

Тепер зробимо зміни ще раз і збережемо.

Історія

$ git log

Щоб переглянути історію змін використовуємо команду git log.

Хостинги для Git репозиторіїв

Добре, з базовими командами розібралися. Тепер поговоримо про те, де зберігати код ваших проектів

Реєструємося на гітхабі

Основні Команди для роботи з віддаленими репозиторіями

$ git remote add [назва репозиторія] [посилання на репозиторій]
$ git push [назва репозиторія] [вітка] 
$ git pull [назва репозиторія] [вітка] 
$ git clone [посилання на репозиторій]
